* LÀM QUEN VỚI GIT/GITHUB, SỬ DỤNG CÁC LỆNH GIT VÀ THỰC HIỆN DEBUGGING CHƯƠNG TRÌNH C/C++.

**I. THỰC HIỆN DEBUGGING MỘT CHƯƠNG TRÌNH C/C++.**

video thực hiện: https://youtu.be/aWwAGi3hjZQ?si=AUlTp7RuQuAnWvNw
- Khi ấn F5 để thực hiện debug, sẽ có những cửa sổ sau hiện ra:
  + Cửa sổ autos: hiển thị những biến và thông tin của biến có trong dòng code mà hiện tại đang debug.
  + Cửa sổ call stack: hiển thị chuỗi các hàm đã được gọi để đến được vị trí hiện tại trong code, giúp truy vết chương trình.
  + Cửa sổ breakpoints: quản lí tất cả breakpoint đã đặt trong code, cho phép đặt/xóa nhiều breakpoint cùng lúc.
  + Cửa sổ output: hiển thị các thông báo từ trình gỡ lỗi, kết quả printf.
  + Cửa sổ diagnostic tools: hiện biểu đồ bộ nhớ, CPU sử dụng, các event trong quá trình debug.
  
**II. LÀM QUEN VỚI GIT/GITHUB, SỬ DỤNG CÁC LỆNH GIT.**
- GIT là 1 hệ thống giúp cho việc quản lý tài liệu, source code...ghi nhớ lại toàn bộ lịch sử thay đổi của source code trong dự án.
- GitHub là một nền tảng dịch vụ trực tuyến dựa trên Git, cho phép lưu trữ, chia sẻ mã nguồn và cộng tác với người khác trên các dự án.
